# AGENTS.md

Guidance for AI coding agents working in this repository.

## What this is

A collection of small, runnable examples of programmatic agent usage. Each
example is a self-contained script demonstrating one feature of an agent SDK.
Most examples are TypeScript; `anthropic-python/` is a Python column. There is
no application to build or deploy — the scripts *are* the product.

## Layout

```
anthropic/              Claude Agent SDK examples
  review.ts             Structured JSON output (JSON Schema enforced)
  review-with-tools.ts  In-process MCP tool that prunes diff noise
  review-with-session.ts Session resume across two passes
  skill-autoload.ts     Agent auto-discovers .claude/skills
  cost-report.ts        Reads cost/token usage from the result message
  utils.ts              readDiff(): stdin or data/<sample>.md
  .claude/skills/       Skills the examples load (e.g. greeting)
ai-sdk/                 Vercel AI SDK 6 examples (assemble-it-yourself agent)
  review.ts             Structured output via ToolLoopAgent + Output.object (Zod)
  review-with-tools.ts  Custom tool() that prunes diff noise (tools is a record)
  review-with-session.ts No built-in session — carry messages[] yourself
  rules-inject.ts       Nothing inherited — read SKILL.md and inject into instructions
  cost-report.ts        Token usage (totalUsage/onStepFinish) + OpenRouter cost
  utils.ts              readDiff(): stdin or data/<sample>.md (own copy)
anthropic-python/       Claude Agent SDK examples in Python (uv-managed)
  review.py             Structured JSON output; model swap + OpenRouter via env=
  review_schema.py      Pydantic models (schema source + validation) + prompt
  utils.py              read_diff(): stdin or data/<sample>.md (own copy)
  pyproject.toml        Deps + uv config (package = false)
data/                   Sample diffs the review examples operate on
```

## Commands

- `npm install` — install dependencies
- `npm run anthropic:review` — run a Claude Agent SDK example (full list in README)
- `npm run aisdk:review` — run the Vercel AI SDK counterpart (needs `OPENROUTER_API_KEY`)
- `npm run anthropic:review -- sample-2` — pass a different sample diff
- `git diff | npm run anthropic:review` — review a real diff from stdin

TypeScript scripts run via `tsx` (ESM, no build step). `npm test` is the npm
placeholder and intentionally exits 1 — there is no test suite yet.

The Python column uses **[uv](https://docs.astral.sh/uv/)**:

- `cd anthropic-python && uv run review.py` — `uv run` syncs `.venv` from
  `pyproject.toml` on first use (no manual venv/pip).
- `uv run review.py sample-2` / `git diff | uv run review.py` — same diff
  sourcing as the TS examples.

## Conventions

- **TypeScript, ESM** (`"type": "module"`), strict mode, `noEmit`. Use
  `import.meta.dirname` for paths, not `__dirname`. The Python column is
  uv-managed; declare deps in `anthropic-python/pyproject.toml`, not `pip`.
- **Code comments are in Polish** — match the existing language and style when
  editing or adding examples. User-facing docs (README) are in English.
- Each example stays **minimal and self-contained**: one file, one feature, no
  shared abstractions beyond `utils.ts`. Don't refactor examples to share code —
  duplication is intentional so each reads top-to-bottom on its own.
- The default model is `claude-sonnet-4-6`. Keep `maxTurns` tight.

## Gotchas

- **Examples make real, billed API calls.** Don't run them in a loop or as part
  of automated checks.
- Auth runs through the local `claude` CLI (Claude Code login); falls back to
  `ANTHROPIC_API_KEY` if set. The `ai-sdk/` examples instead need
  `OPENROUTER_API_KEY` (the assemble-it-yourself agent has no implicit auth).
- `anthropic/cost.json` and `ai-sdk/cost.json` are generated by the `:cost`
  scripts and are gitignored — never commit them.
- The `ai-sdk/` examples target **AI SDK 6** (`ai@^6`); the OpenRouter provider
  peers on `ai@^6`, so do not bump `ai` to the v7 beta.
- Custom (in-process MCP) tools require streaming-input mode: the `prompt` must
  be an async generator, not a string (see `review-with-tools.ts`).
